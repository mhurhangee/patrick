# living-docs

Working documents with a **finite lifespan** — execution plans, migration
checklists, in-flight design notes. They live here while a piece of work is
active and are **deleted when it ships** (the durable record is the code, the
git history, `CLAUDE.md`, and the memory index).

Keep the repo root clean: planning docs go here, not at the root. Durable,
load-bearing docs (`CLAUDE.md`, `CONTRIBUTING.md`, `BRAND.md`) stay at the root.
`IDEAS.md` (gitignored scratch backlog) is separate — that's an open-ended idea
list, not a per-task plan.
