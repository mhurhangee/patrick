# Deployment

PatrickOS runs in three modes. The same software, different configurations.

## Local

The desktop app bundles everything — UI, API server, and a local SQLite database. Install it and open it. No configuration required.

For AI inference, install [Ollama](https://ollama.com) and pull a model:

```bash
ollama pull llama3.2
```

PatrickOS detects Ollama automatically on startup.

## Self-hosted

Run the API and frontend on your firm's own infrastructure. Each firm gets an isolated database.

```bash
git clone https://github.com/mhurhangee/patrickos
cd patrickos
pnpm install

# Configure environment
cp .env.example .env
# Set DATABASE_URL and AI provider keys

# Build and run
pnpm --filter api build
pnpm --filter api start
pnpm --filter web build   # serve dist/ with your preferred static host
```

The API server is a single compiled binary — deploy it anywhere that runs an executable.

For the database, set `DATABASE_URL` to a [libSQL](https://github.com/tursodatabase/libsql) connection string. Self-hosted firms can run `sqld` (the open-source libSQL server) on their own infrastructure.

## Cloud

Hosted by PatrickOS. Your firm's data is stored in an isolated database in your chosen region (EU or US).

Get in touch to set up a cloud account.

## Environment variables

| Variable | Description |
|---|---|
| `VITE_API_URL` | API server URL (frontend) |
| `DATABASE_URL` | libSQL connection string |
| `ANTHROPIC_API_KEY` | Anthropic API key (cloud AI) |
| `OPENAI_API_KEY` | OpenAI API key (cloud AI) |
