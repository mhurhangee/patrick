# Frontend

The frontend lives in `apps/frontend`. It's a React app built with Vite.

## Running it

```bash
pnpm dev      # dev server on localhost:5173
pnpm build    # build to dist/
```

Or from the workspace root: `pnpm dev` starts frontend + API together.

## Stack

- **React 19** — UI library
- **Vite** — build tool and dev server
- **TanStack Router** — client-side routing (file-based, fully typed)
- **Tailwind CSS v4** — utility-first CSS
- **shadcn/ui** — pre-built components built on Radix UI primitives
- **Biome** — linter and formatter

## Routing

TanStack Router uses file-based routing — the file structure in `src/routes/` maps directly to URL paths:

```
src/routes/
├── __root.tsx           Layout wrapper
├── index.tsx            → /            (landing page)
├── docs.tsx             → /docs        (docs layout with sidebar)
├── docs/
│   ├── index.tsx        → /docs/       (redirects to /docs/architecture)
│   ├── architecture.tsx → /docs/architecture
│   └── ...
└── workspace/
    └── index.tsx        → /workspace/
```

TanStack Router generates `routeTree.gen.ts` automatically when you run the dev server. Don't edit that file.

## Pages

### Workspace (`routes/workspace/index.tsx`)

The main app. A three-panel layout:

```
[ Sidebar ] | [ Asset viewer ] | [ Chat panel ]
```

**Sidebar** — project selector at the top (opens a full CRUD sheet), assets grouped by type below. All data is real — loaded from and persisted to the API.

**Asset viewer** — tabs for open assets, split view support (two side by side). Each asset has a collapsible metadata panel (title, type, date, notes) and a content area. Sources will render a PDF viewer; artifacts will render a Tiptap rich text editor. Both are currently placeholders.

**Chat panel** — collapsible. Message thread and input. Currently mock — messages are hardcoded and sending only updates local state.

### What's real vs mock

| Thing | Status |
|---|---|
| Projects | **Real** — full CRUD via API |
| Assets (sources + artifacts) | **Real** — full CRUD via API |
| Asset metadata (title, type, date, notes) | **Real** — persisted via API |
| Asset content (PDF viewer / editor) | **Placeholder** — coming next |
| Chat | **Mock** — local state only |

## API client (`lib/api.ts`)

All HTTP calls go through `lib/api.ts`:

```typescript
const BASE_URL = import.meta.env.VITE_API_URL ?? "http://localhost:3000"

export const api = {
  projects: { list, create, rename, delete },
  assets:   { list, create, update, delete },
}
```

`VITE_API_URL` is a Vite env variable — set it in `.env` to point at a different API. Defaults to `localhost:3000`.

## Component library

Components in `components/ui/` come from [shadcn/ui](https://ui.shadcn.com). They're copied into source (not installed as a package) so you can edit them directly.

Always use `<Button>` from shadcn — never a plain `<button>` element. This keeps styling, sizing, and accessibility consistent.

## Docs pages

Each doc page imports an MDX file and renders it with Tailwind's `prose` class. The sidebar is driven by `lib/doc-nav.ts`. To add a page: write the MDX in `src/content/docs/`, create the route file in `src/routes/docs/`, add an entry to `DOC_NAV`.
