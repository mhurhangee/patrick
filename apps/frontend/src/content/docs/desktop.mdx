# Desktop

The desktop app lives in `apps/desktop`. It's a [Tauri](https://tauri.app) v2 application.

## What Tauri does

Tauri wraps a web frontend in a native desktop window using the OS's own browser engine — WebKit on Linux/macOS, WebView2 on Windows. This makes binaries much smaller than Electron (no bundled Chromium).

The app shell and system integration are written in Rust. Everything the user sees is React.

## Structure

```
apps/desktop/
└── src-tauri/
    ├── src/
    │   ├── lib.rs       App setup — sidecar startup, zoom fix
    │   └── main.rs      Entry point
    ├── binaries/        Compiled API binary lives here (gitignored)
    ├── capabilities/    Permissions the app is granted
    ├── icons/           App icons
    ├── Cargo.toml       Rust dependencies
    └── tauri.conf.json  Tauri configuration
```

## Running

```bash
# From workspace root:
pnpm build:api      # build API binary and copy to binaries/
pnpm dev:desktop    # start Tauri dev (also starts frontend dev server)
```

`build:api` must run before `dev:desktop` — the API sidecar is a compiled binary, not a source-watched process.

## The sidecar

A sidecar is an external binary that Tauri bundles and manages alongside the main app. The API (Hono on Bun, compiled to a standalone binary) runs as a sidecar.

The binary must be in `src-tauri/binaries/` and named with the Rust target triple:

```
binaries/api-x86_64-unknown-linux-gnu   # Linux x64
binaries/api-aarch64-apple-darwin        # macOS Apple Silicon
```

`pnpm build:api` from the workspace root handles building and copying automatically (hardcoded to Linux x86_64 for now).

The binary is registered in `tauri.conf.json`:
```json
"bundle": { "externalBin": ["binaries/api"] }
```

## lib.rs

`src/lib.rs` does three things:

**1. Starts the API sidecar:**
```rust
let data_dir = app.path().app_data_dir()?;
std::fs::create_dir_all(&data_dir)?;
let db_path = data_dir.join("patrickos.db");

app.shell().sidecar("api")?
    .env("DATABASE_URL", format!("file:{}", db_path.display()))
    .env("PORT", "3000")
    .spawn()?;
```

The database lives in the OS app data directory:
- Linux: `~/.local/share/com.patrickos.desktop/`
- macOS: `~/Library/Application Support/com.patrickos.desktop/`

**2. Applies a zoom fix for WSL2/Linux DPI scaling:**
```rust
let window = app.get_webview_window("main")?;
window.set_zoom(1.5)?;
```

WSLg/X11 reports 96 DPI regardless of actual display scale, causing WebKitGTK to render at a tiny size on high-DPI displays. The 1.5× zoom corrects this.

**3. Sets up logging in debug builds:**
```rust
if cfg!(debug_assertions) {
    app.handle().plugin(tauri_plugin_log::Builder::default()
        .level(log::LevelFilter::Info).build())?;
}
```

Log output appears in the terminal where you ran `pnpm dev:desktop`.

## Capabilities

`capabilities/default.json` is Tauri v2's allow-list — nothing works unless explicitly listed:

```json
{ "permissions": ["core:default", "shell:allow-execute", "shell:allow-spawn"] }
```

`shell:allow-spawn` is what permits starting the API sidecar.

## Building a release

```bash
pnpm build:api
pnpm --filter desktop build
```

Produces an installer in `src-tauri/target/release/bundle/`. Build targets are `deb` and `rpm` (AppImage excluded — requires `linuxdeploy` which isn't available in WSL2).
